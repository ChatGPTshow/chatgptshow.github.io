最新模型GPT-5.6来了，Codex和GPT正式合并，附如何升级体验到GPT-5.6教程

北京时间7月10日凌晨，GPT-5.6正式开放。谁能用、强了多少、怎么升级，这篇一次讲清。

前几天的文章里，我们还在聊 ChatGPT 换成蓝底黄标，Sol、Terra、Luna 是不是真要来了。

ChatGPT变色了，GPT-5.6这次真有点像要来了open in new window

结果没让大家等太久。

美国当地时间7月9日，也就是北京时间7月10日凌晨，OpenAI 正式发布 GPT-5.6，并开始向 ChatGPT、Codex 和 API 用户推送。之前的 limited preview 已经结束，这次是正式开放！

而且，这次不只是换一个模型。ChatGPT 的桌面应用、Codex 和新的 Work 模式一起改了。

一、GPT-5.6到底发了什么

GPT-5.6 不是单一模型，而是三个档位：

  • Sol 是旗舰模型，适合复杂推理、编程、科研、网络安全和长链路任务。
  • Terra 强调平衡，定位是以更低成本提供接近 GPT-5.5 的综合能力，更适合当日常主力。
  • Luna 速度更快、价格更低，适合高频小任务、快速改写、批量处理和对延迟敏感的工作。

用一句话讲：Sol 负责啃硬骨头，Terra 负责大部分日常活，Luna 负责把高频工作跑得更快。

API 定价也直接体现了这个分层。每百万 token 的输入/输出价格分别是：Sol 5/30 美元，Terra 2.5/15 美元，Luna 1/6 美元。

二、GPT-5.6比5.5强在哪里？

先说结论：如果只看 Sol，它对 GPT-5.5 的提升不是某一科偏科，而是编程、专业工作、电脑操作、医疗和网络安全一起往上走。

下面这组数据都来自 OpenAI 7月9日的官方发布页，对比的是同一张评测表中的 GPT-5.6 Sol 和 GPT-5.5：

编程方面还有两个数字很有代表性。GPT-5.6 Sol 在 Artificial Analysis Coding Agent Index 中拿到 80 分，GPT-5.5 是 76.4；在 Terminal-Bench 2.1 中，Sol 的常规成绩是 88.8%，开启 Ultra 后到 91.9%，GPT-5.5 是 85.6%。

官方合作伙伴给出的体感也更贴近实际工作。Qodo 的代码评审测试中，GPT-5.6 用约三分之一的 token 超过了 GPT-5.5，中位延迟约降一半。而Lovable 的应用构建流程里，步骤数减少约 25%，工具调用少 35%至48%，卡住的运行下降 15%。

但有一点要说清楚：这不等于 Sol、Terra、Luna 每一项都能赢 GPT-5.5。例如 Luna 在 Big Finance Bench 上是 36%，GPT-5.5 是 49%。Luna 的任务本来就不是“无脑替代旗舰”，而是用更低价格和更快速度处理合适的任务。

我觉得这才是 5.6 这次真正的重点：不只把上限抬高，也开始认真解决“这个任务到底值得用多少算力”。

三、模型之外，还有哪些变化？

1. Codex应用并入ChatGPT桌面应用

OpenAI 官方的原话是，Codex app 开始与新的 ChatGPT desktop app 合并。

老 Codex 用户正常更新后，就会变成新版 ChatGPT 桌面应用，不过你可以选择保留Codex应用图标。

而原来的 ChatGPT 桌面应用则会被改名为 ChatGPT Classic。

不过，这不是把 ChatGPT 和 Codex 粗暴地捣成一个按钮。新应用里仍然有明确分工：

模式适合做什么
Chat普通问答、讨论、写作、改稿，你和它一轮轮对话
Work把一个目标交给它，让它连接文件、应用和浏览器，持续工作几十分钟甚至几小时，直接交付表格、文档、PPT 或网站
Codex专注编程和技术工作，包括读代码、修复问题、运行测试、评审 PR 和管理仓库

所以，网页版里出现的 Chat 和 Work，可以简单理解为：Chat 是“陪你想”，Work 是“帮你做完”。

同样，原Codex软件中也出现了相关的Codex和Work，点击左上角图标即可切换

Work 内置了 Codex 技术，但官方帮助文档仍把 Work 和 Codex 称为两种不同体验。更准确地说：入口开始合流，模型和基础能力开始共用，但用途还是分开的。

说白了，入口合并了，活还是各干各的。

不过以后就没有Codex app了，一个时代结束了，顶着Codex的图标的ChatGPT app，说实话还挺别扭的哈哈哈。

2. Work和Codex开始共用一个额度池

OpenAI 现在把 Codex、ChatGPT Work、ChatGPT for Excel 和 Workspace Agents 的消耗算在同一个 agentic usage and credit pool 里。

这点还是很重要的。你在 Work 里跑一个长任务，可能会吃掉 Codex 的可用额度；反过来也一样。而且 Sol、max 和 ultra 这类更高思考强度，本来就会消耗更多。

发布前后，确实有一批 Codex 用户发现自己的额度被重置或回补，也有人没有收到，甚至遇到额度显示来回跳动。目前 OpenAI 没有发布“GPT-5.6 上线即为全员清空用量”的统一公告。官方能确认的是，部分 Plus 和 Pro 用户有 banked reset，也就是可以主动使用的“储存重置”。

说实话，这块现在有点乱。同样的套餐，有人突然满额，有人什么都没收到，还有人看着进度条来回跳。拿不准自己到底还能跑多少，确实挺让人心里没底的。

所以，如果你看到额度回来了，可以趁机试 5.6，但别把这理解成长期提额，也别一上来就用 Sol Ultra 跑无关紧要的小任务。

3. Claude也在同期重置了部分用户额度

更有意思的是，GPT-5.6 发布后，**ClaudeDevs 官方账号也发文确认，已为所有用户重置 5 小时和每周用量。**用户反馈显示，这次重置也包括 Fable 5 额度。

不过也从这一行为可以看出GPT-5.6的正式开放确实对Anthropic带来了不小的压力。模型强不强，看同行竞争对手的反应就对了。

同期额度重置，至少说明大家都很在意这批高频用户。

四、什么人可以用GPT-5.6？

这次和之前的小范围预览完全不同。按 OpenAI 7月9日公布的规则：

用户类型可用范围
Free、Go在 ChatGPT Work 和 Codex 中可使用 GPT-5.6 Terra
PlusChat 中可在 medium 及以上思考强度使用 Sol；Work 和 Codex 可选 Sol、Terra、Luna，Codex 可用 ultra
Pro拥有 Plus 的范围，还可在 Chat 里选 Sol Pro，在 Work 里使用 ultra
Business、Enterprise可在 Chat 使用 Sol，在 Work 和 Codex 选择三个模型；Enterprise 可用 Sol Pro 和 Work ultra
API开发者可调用 Sol、Terra、Luna，按 token 单独计费，不需要为了 API 另行购买 ChatGPT 订阅

Edu 用户的 Work 正在滚动开放，但具体模型选项受组织管理员、地区和推送批次影响，以账号内实际显示为准。

有些人会在这里有误解。看到 Work 入口，不等于你已经拿到所有 5.6 模型；有入口没有 Sol，或者桌面端有、网页端还没有，发布初期都很正常。别为了一个还在灰度中的按钮瞎折腾账号。

max 思考强度对所有已获得 GPT-5.6 Work/Codex 权限的用户开放，可以在设置里打开。ultra 会默认调度四个智能体并行工作，结果更强，但用量消耗也会明显增加。

如何升级体验到GPT-5.6

Free/Go 账号可以先用 Terra。

这是当前成本最低的试用方法。不过免费计划的额度会比付费计划少，适合先判断体感，不适合长时间跑大项目。

如果需要更多额度，可以按照自身需求升级Plus或Pro（文末有升级教程）

Plus 用户可以在 Work 和 Codex 中选 Sol、Terra、Luna，Chat 里把思考强度调到 medium 或更高后可用 Sol，Codex 还能打开 ultra。

如果你是个人用户，主要写作、做研究、处理文件，偶尔跑代码，Plus 通常已经够用。升级后别默认所有任务都用 Sol，日常活先用 Terra，真正的硬任务再换 Sol，额度会耐用很多。

而 Pro 的价值主要在 Sol Pro、Work ultra 和更高的可用量。如果你每天都用 AI 跑长任务，或者会直接用它交付代码、研究报告、表格和演示文稿，Pro 才更容易回本。

只是偶尔问几个问题，不建议为了“能看到最强选项”就直接上 Pro。
