# 注意，Cursor开始封禁账号：提示User is unauthorized，Unauthorized request

最近Cursor开始封禁账号了，客户端会出现提示Unauthorized request， User is unauthorized

Pro账号在尝试开关Enable usage-based pricing会提示 User is unauthorized

官方论坛也有不少用户反馈此类问题的

对此Cursor社区开发者也对其进行了解释👇

对于那些Cursor被封禁账号的用户来说，本次封禁主要包含三个原因

  1. 帐户是从第三方购买的——大多数此类帐户都是使用被盗银行信息设置的，Cursor官方现在正在检测和阻止这些帐户。因此，不是直接从 Cursor 购买的账户将被阻止。

  2. 使用的是临时电子邮件地址——由于使用临时电子邮件地址设置的帐户存在滥用行为，Cursor官方正在阻止使用已知临时电子邮件地址的帐户的请求。需要使用标准电子邮件地址提供商，例如@gmail.com。

  3. 使用的是 VPN —— 同样是由于滥用，官方不得不阻止某些 VPN 提供商访问 Cursor 服务。虽然Cursor官方不认为 VPN 本质上是坏的，但他们必须对某些滥用行为频发的提供商实施封锁，以确保 Cursor 服务不会分发给普通用户。

如果你觉得这些你都没有违反的话，且认为没有理由进行屏蔽，那可以前往社区该贴下发布请求 ID，社区会逐一进行回复。
