前期必须准备及注意事项

安装Node.js

由于Claude Code和Codex 是用 Node.js 开发的，所以我们需要安装Node.js环境，如果你已经安装 Node.js，请确保版本为 18.0 或更高版本的要求。

可以前往官网下载：https://nodejs.org/en/download/

安装教程很简单，不要在C盘安装即可，网上也有很多教程。

Linux、Mac可以直接通过以下的方式安装（一行一句命令）：

curl -fsSL https://deb.nodesource.com/setup_22.x | sudo bash -
sudo apt-get install -y nodejs

确认是否安装成功。

node --version
npm --version

出现具体的版本信息则代表环境安装成功

  • 重要注意事项
    • 需要确保Node.js为18.0 或更高版本
    • 环境变量重启：设置环境变量后请务必重启命令行窗口

Claude Code安装部署

Claude Code安装部署有以下方法：

  • Gac站点的cc（不需要魔法）
  • Anthropic官网的cc（需要魔法）

大家可以按照自己的需求来进行部署安装，这里推荐安装Anthropic官网的cc，通过配置api来使用，灵活性会更高一些

一、部署Gac站点的cc

官网文档：https://gaccode.com/install-claude-code

使用 npm 在您的系统上全局安装 Claude Code，-g 标志将 Claude Code 全局安装，使其可从任何目录使用。

npm install -g https://gaccode.com/claudecode/install --registry=https://registry.npmmirror.com

检查 Claude Code 是否安装成功

claude --version

当您首次使用时，GAC要求您验证订阅。您可通过出现的浏览器窗口登录以完成验证

输入claude 之后授权即可使用

  • 此过程不要使用代理
  • 如果您使用远程服务器，访问命令行中的链接并输入验证码以完成验证
  • 如果有网络问题可以输入 claude --pick-relay 切换接入点
  • 要升级 Claude Code 到最新版本，只需运行相同的安装命令
npm install -g https://gaccode.com/claudecode/install --registry=https://registry.npmmirror.com

定期更新可确保您获得最新功能、性能改进和错误修复。

二、部署官方的cc（推荐）

官网文档：https://claude.com/product/claude-code

npm install -g @anthropic-ai/claude-code

安装完成后，使用claude –version验证是否安装成功

注意：如果没有安装git会显示

安装成功之后，就是要配置GAC站点的api_key和api_url，Windows系统和Linux/Mac系统不同，可按照自身情况进行配置

（1）Windows系统

首先，win+R，输入sysdm.cpl（或者左下角直接搜索 "编辑系统环境变量"）

在 “高级”中找到环境变量，在"用户变量"区域点击"新建"

设置api_url的环境变量，变量名为：ANTHROPIC_BASE_URL

变量值为：https://gaccode.com/claudecode

接下来去站点新建一个密钥

创建好后将新建的密钥，添加到新的环境变量中，变量名为：ANTHROPIC_API_KEY

依次点击确认，关闭所有窗口

重新打开一个命令提示符cmd，运行以下命令，正常显示则说明配置成功

echo %ANTHROPIC_BASE_URL%
echo %ANTHROPIC_API_KEY%

输入claude 如果有出现key 和url则证明配置成功，可正常运行

（2）Linux/Mac系统

设置环境变量（永久生效）

# 打开文件
nano ~/.bashrc

# 在末尾加入一行
export ANTHROPIC_BASE_URL=https://gaccode.com/claudecode
export ANTHROPIC_API_KEY=你的完整API密钥

保存退出后执行：

source ~/.bashrc

这样就对当前用户永久生效。 验证查看配置，有则说明配置成功

echo %ANTHROPIC_BASE_URL
echo %ANTHROPIC_API_KEY

输入claude 如果有出现key 和url则证明配置成功，可正常运行

不少朋友在输入Claude之后可能会出现下面这样的报错，这个是Anthropic对部分地区的用户进行地区限制。

可以尝试切换代理的节点国家，或者执行命令绕过

node --eval "const os = require('os'); const fs = require('fs'); const path = require('path'); const homeDir = os.homedir(); const filePath = path.join(homeDir, '.claude.json'); if (fs.existsSync(filePath)) {const content = JSON.parse(fs.readFileSync(filePath, 'utf-8')); fs.writeFileSync(filePath, JSON.stringify({ ...content, hasCompletedOnboarding: true }, null, 2), 'utf-8');} else {fs.writeFileSync(filePath, JSON.stringify({ hasCompletedOnboarding: true }), 'utf-8');}"

重新进入claude即可正常使用

Codex 安装部署

Codex安装部署同样有以下方法：

  • Gaccode站点的codex（不需要魔法）
  • OpenAI官网的codex（需要魔法）（推荐）

按需安装即可，这里同样推荐安装OpenAI官网的Codex，通过配置api来使用，灵活性会相对较高一些

一、部署Gac站点的codex

官网文档：https://gaccode.com/install-codex

npm install -g https://gaccode.com/codex/install

当您首次使用时，GAC会要求你进行验证订阅。你可通过出现的浏览器窗口登录以完成验证

输入codex 之后授权即可使用

  • 此过程不要使用代理
  • 如果您使用远程服务器，访问命令行中的链接并输入验证码以完成验证

二、部署官方的codex（推荐）

官网文档：https://developers.openai.com/codex/cli

npm install -g @openai/codex

安装完成之后，在~/.codex/config.toml 文件中添加以下配置：

Windows的路径是 C:\Users<用户名> 下找到.codex目录

model_provider = "gac"
model = "gpt-5"
model_reasoning_effort = "high"
disable_response_storage = true
preferred_auth_method = "apikey"

[model_providers.gac]
name = "gac"
base_url = "https://gaccode.com/codex/v1"
wire_api = "responses"

在~/.codex/auth.json 文件中配置API密钥:

{
    "OPENAI_API_KEY": "你的API密钥"
}

使用与 Claude Code 相同的 API密钥即可，格式如 sk-xxxxxxxxxx 之后重开一个新窗口输入codex 即可
